# TSAM Mapping — Repository Components

This mapping explains how this repository instantiates TSAM layers. It is informative, but the bindings between claims and evidence are normative.

| TSAM Layer | Repository component(s) |
|------------|--------------------------|
| Governance Semantics | terminology, policies, lifecycle model |
| Assurance Levels | AL1–AL4 definitions, matrices, upgrade guidance |
| Conformance Verification | verifier workflows and validation guidance (where present); cross-repo CTS alignment |
| Runtime Integrity Controls | standards alignment guidance; control objective mappings |
| Evidence & Observability | evidence bundles, structured outputs, machine-readable artifacts |

This repository MUST NOT be interpreted as the only possible TSAM implementation. TSAM is registry-agnostic and may be realized in other architectures.
