# Public-Sector Authoritative Directory

## Business problem

A public-sector operator publishes a directory whose claims may affect access to services or recognition. The release pack emphasizes authority, freshness, redress, and auditability.

## Who needs assurance?

- Ecosystem operator.
- Registry operator.
- Relying party.
- Assessor or auditor.
- Affected user or consumer.

## Evidence required

- CTS scenario profile report.
- TSPP posture report with relying-party controls.
- Combined Assurance Manifest.
- Public Assurance Summary.
- Limitations and contact/redress metadata.

## Adoption impact

This use case converts trust from a narrative claim into an executable onboarding and review process.
