# Ecosystem Registry Onboarding

## Business problem

An ecosystem operator evaluates whether a registry may participate in shared governance infrastructure. The evidence package supports admission, renewal, suspension, and revocation decisions.

## Who needs assurance?

- Ecosystem operator.
- Registry operator.
- Relying party.
- Assessor or auditor.
- Affected user or consumer.

## Evidence required

- CTS scenario profile report.
- TSPP posture report with relying-party controls.
- Combined Assurance Manifest.
- Public Assurance Summary.
- Limitations and contact/redress metadata.

## Adoption impact

This use case converts trust from a narrative claim into an executable onboarding and review process.
