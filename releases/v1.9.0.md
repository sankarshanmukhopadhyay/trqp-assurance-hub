# TRQP Assurance Hub v1.9.0 — End-to-End Assurance Evidence Chain

## Summary

This release replaces metadata-only cross-repository smoke testing with authentic CTS and TSPP evidence execution, validation, composition, and fail-closed assurance decisions.

## Coordinated release tuple

| Component | Version | Role |
|---|---:|---|
| TRQP Assurance Hub | v1.9.0 | Evidence composition and assurance decision |
| TRQP Conformance Suite | v1.6.0 | Protocol conformance evidence |
| TRQP-TSPP | v0.14.0 | Security and privacy posture evidence |

## Added

- Canonical machine-readable compatibility registry.
- Shared project-status governance contract.
- Assurance-decision schema and traceability report.
- Fail-closed run and target identity validation.
- Negative tests for mismatched evidence.
- Reproducible end-to-end workflow and evidence artifact upload.

## Changed

- The combined-assurance workflow now executes producer repositories and consumes authentic reports.
- Producer evidence is generated with common identity before checksums are recorded; reports are no longer mutated after evidence indexing.
- Current documentation and release metadata now consistently identify v1.9.0 / v1.6.0 / v0.14.0.

## Compatibility

Release set `ots-2026-07` is the canonical supported tuple. Historical matrices remain informative only.

## Validation

```bash
make validate
make assurance-check
```

The end-to-end workflow must also pass in GitHub Actions before tagging.

## Known result

The current CTS fixture evidence contains failing assertions, so the composed assurance decision is correctly `fail`. This demonstrates fail-closed behaviour but is not a positive conformance demonstration.

## Authority and limitations

The Hub owns evidence ingestion, composition, assurance evaluation, and publication. It does not own the TRQP protocol specification or external accreditation and certification decisions.

## Upgrade guidance

Use the canonical release tuple, execute both producers with the same `TRQP_RUN_ID` and `TRQP_TARGET_ID`, then compose their reports with `tools/run_combined_assurance.py`.
