# TRQP Assurance Hub v1.0.0

This is the first stable release of the Assurance Hub. The AL1–AL4 assurance contract
is frozen, the canonical AL definitions SHA is unchanged, and the toolset is validated
against the coordinated CTS v0.8.0 + TSPP v0.6.0 release train.

## Highlights

- **Ayra Trust Network mapping** (`tools/ayra-mapping.md`): end-to-end crosswalk from
  the five canonical Hub controls to Ayra conformance tiers. Covers evidence artifact
  requirements, sufficiency notes per AL, and a three-tier submission checklist
  (Basic / Cross-Ecosystem / Sovereign). Ayra operators can use this as a pre-submission
  checklist before engaging the Ayra conformance process.
- **README stabilized**: version references now match `VERSION` and `al-contract.json`
  throughout. The v0.9.0 README/VERSION divergence is resolved.

## What "v1.0.0" signals

The Hub's canonical AL definitions (`docs/guides/assurance-levels.md`, SHA pinned in
`al-contract.json`) are treated as frozen for downstream consumption:

- Downstream repos (`TRQP-TSPP`, `trqp-conformance-suite`) MUST pin to v1.0.0 for
  audit-stable deployments.
- Breaking changes to AL semantics will increment the major version.
- Additive guidance documents (like `tools/ayra-mapping.md`) are minor increments.
