# v1.5.0

## Summary

This release hardens the Operational Stack evidence chain. Combined Assurance Manifest generation now fails closed when CTS and TSPP do not agree on shared execution identity, and the checked-in demo bundle is validated end to end in CI.

## What changed

- Added fail-closed `run_id` / `target_id` validation to `tools/generate-manifest.py`
- Added `tools/validate_operational_stack.py` for bundle-level integrity checks
- Extended `tools/validate_examples.py` to validate `artifacts/operational-stack/`
- Regenerated checked-in operational stack artifacts with aligned identifiers and current versions
- Refreshed docs and cross-repo compatibility references
