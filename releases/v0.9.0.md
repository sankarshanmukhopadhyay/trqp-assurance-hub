# Release v0.9.0

## Summary

Schema alignment and CI hardening. No changes to AL semantics, assurance profiles, or
guidance documents.

## Fixed

- **`schemas/combined-assurance-manifest.schema.json`** — schema now matches the actual output
  of `tools/generate-manifest.py`. The previous schema defined a flat root shape
  (`build_id`, `target`, `version_tuple`) with `additionalProperties: false`, while the tool
  emits a nested document (`build: {build_id, target, commit, ci_run_url}`, `tools: {...}`,
  `generator: {...}`). The mismatch caused the `combined-assurance-smoke` CI to fail on schema
  validation for every push to `main`.

  Artifact items are also corrected: `produced_by`, `media_type`, `notes`, and `format` are
  now declared in the schema and are no longer blocked by `additionalProperties: false`.

- **`examples/combined-assurance-manifest.example.json`** — updated to the corrected schema
  shape so `tools/validate_examples.py` passes.

## Added

- **`validate` job in `.github/workflows/quality.yml`** — runs `tools/validate_examples.py`
  and `scripts/doc_tests.py` on every push and pull request, catching example/schema drift
  and JSON/YAML parse regressions before they land.

## Cross-repo

Downstream repos (`trqp-conformance-suite`, `TRQP-TSPP`) updated their `al-contract.json`
pins from `v0.8.1` to `v0.9.0` and wired `scripts/doc_tests.py` into their CI pipelines
as part of this coordinated commit.

## Compatibility

- AL semantics: **unchanged** (canonical SHA pinned in `al-contract.json` is unchanged)
- Assurance profiles: **unchanged**
- `tools/generate-manifest.py` CLI: **unchanged**
- `schemas/combined-assurance-manifest.schema.json`: **breaking change to schema shape** —
  consumers validating against the old flat schema must adopt the new nested shape.
  The `artifacts_test/manifest.json` fixture already conformed to the new shape.
