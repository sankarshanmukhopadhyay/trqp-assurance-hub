# TRQP Assurance Hub v1.0.1

Patch release. CI fix and Ayra mapping update aligned to the full Ayra TRQP Profile
v0.5.0-draft spec. No changes to AL semantics, `al-contract.json`, or canonical SHA.

## Changes

- **CI fix** (`releases/v1.0.0.md`): added `text` language specifier to the SHA
  fenced code block, resolving MD040 markdownlint failure on the `quality` workflow
  (`#73`).
- **`tools/ayra-mapping.md` updated**: crosswalk now reflects the complete Ayra TRQP
  Profile v0.5.0-draft requirements including:
  - `did:webvh` MUST requirement for all ecosystem, trust registry, and cluster
    identifiers — now documented as normative across all five Hub controls with a
    clear manual-check flag until automated validation ships.
  - JWS response signing reclassified as MUST at Ayra Basic tier (not an AL2 upgrade)
    to match the Ayra Profile normative text.
  - All eight Ayra extension endpoints (`/metadata`, `/entities/...`, `/ecosystems/...`,
    `/lookups/...`) added to the CTRL-01 and CTRL-03 evidence tables with CTS test
    case cross-references.
  - Submission checklist updated: cross-ecosystem tier now includes recognition security
    test reference (`test_10_recognition_security.py`) and `evidence_retention_days`
    thresholds (≥ 90 Basic, ≥ 365 Sovereign).
  - "Known gaps" section expanded: `did:webvh` format validation, recognition chain
    depth, EGF endpoint resolution, and network enrollment are explicitly called out
    as manual checks.
