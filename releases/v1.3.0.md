# v1.3.0

Determinism Foundation — Release Set A.

## Included

- **`generate-manifest.py --dry-run`**: validates all inputs and emits a manifest preview to stdout without writing any output files. Schema validation is still applied in dry-run mode. This makes it safe to add as a pre-merge check in downstream repos. Added to CI (`quality.yml`) as a first-class validation step on every push to `main`.
- **`run_operational_stack.py` audit trail**: the metadata JSON written by the operational stack runner now includes a `generated_at` RFC 3339 timestamp, completing the operational stack's audit record.
- **CI `validate` job**: `quality.yml` now includes a `validate` job running `tools/validate_examples.py`, `scripts/doc_tests.py`, and `generate-manifest.py --dry-run` on every push and pull request.
- **Version sync**: downstream cross-repo references updated to TRQP-TSPP v0.9.0 and Conformance Suite v1.1.0.
