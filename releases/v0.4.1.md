# Release Notes — v0.4.1

This release hardens cross-repository interoperability by making Assurance Levels (AL1–AL4) machine-pinable and audit-stable.

## Highlights
- Adds `al-contract.json` to publish a machine-readable contract for canonical AL semantics (includes SHA-256 of the normative AL doc).
- Clarifies normative status of `docs/guides/assurance-levels.md` for downstream consumption.

## Upgrade notes
- Downstream repos should reference `al-contract.json` (or the canonical AL doc hash) to prevent semantic drift across releases.
