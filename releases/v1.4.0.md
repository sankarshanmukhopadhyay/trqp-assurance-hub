# v1.4.0

Adopter Surface and Coverage Completion — Release Set B.

## Included

### Minimal viable bundle templates (AL1 and AL2)
`examples/al1-evidence-bundle/` and `examples/al2-evidence-bundle/` provide
structured starting points for adopters who want guidance without the full
AL3/AL4 overhead. Each includes a `README.md` (what to produce, what to omit,
and a complete toolchain invocation sequence), a `version-tuple.json`, a
`combined-assurance-manifest.json`, and an `operator-declaration.md` with
omission rationale.

### Audit companion guides (AL3 and AL4)
`examples/al3-evidence-bundle/AUDIT_GUIDE.md` and
`examples/al4-evidence-bundle/AUDIT_GUIDE.md` map every evidence artifact in
each bundle to its verifier check, expected outcome, and failure signal. Both
guides end with a numbered AL pass criteria checklist.

### Adopter workflow template
`templates/combined-assurance-workflow.yml` is a copy/paste GitHub Actions
workflow that runs CTS, the TSPP harness, and `generate-manifest.py` against
any registry — with version pins, dry-run validation, and artifact upload
steps — without requiring a Hub repository dependency.

### Schema contract
`schemas/CONTRACT.md` declares Stable, Extensible, and Experimental stability
tiers for every schema in `schemas/`. Includes per-schema field stability
tables, enum freeze policy, and the AL contract pin upgrade checklist for
downstream adopters.

### AL decision tree
`docs/guides/al-decision-tree.md` guides adopters from deployment context
(consequence → external reliance → continuous operation) to the correct AL,
with a summary table, per-AL quick reference (what to produce, what to omit,
toolchain invocation), and upgrade path guidance.

### Hub ↔ CTS ↔ TSPP crosswalk
`docs/reference/hub-cts-crosswalk.md` provides a three-way mapping from each
Hub control ID to the CTS test IDs that exercise it and the TSPP requirement
IDs that assert it, with a reverse index in both directions and a coverage
gap section explaining which controls are TSPP-only (not exercisable by CTS).

### Extended example validation
`tools/validate_examples.py` now validates JSON files in all AL evidence bundle
directories (`examples/al*-evidence-bundle/`) against their paired schemas, in
addition to the existing flat `examples/*.example.json` coverage.
